OOP - Abstracting & Naming Concepts: Level 1 Editorial

Being a good developer is essentially two parts:

  • Problem solving skills
  • Coding skill
How do you approach a large and complex problem?


  • A part of problem solving skills is how you break down a problem. A problem that is large and complex at first, can be broken down into small manageable chunks.
  • To break down a problem, identify small parts which can be solved independently from others. Identify implicit concepts in the problem(could be generic or very domain-specific). Give example:
    Hint: Look for nouns in the problem statement
    Some of them will be good cases for enums
  • Naming: What's in a name- Everything! Good naming is an indicator, that you have understood and broken down the problem well. Name should be meaningful and should abstract the concept you are trying to describe. The naming should trickle down to your code as well, e.g., classes, methods, variables, namespaces etc.

    class TelecomUser:
    
    def __init__(self, userName, msisdn, initialBalance, circle):
        self.userName = userName
        self.msisdn = msisdn
        self.balance = initialBalance
        self.circle = circle
    
    def getMsisdn(self):
        return self.msisdn
    
    def getUsername(self):
        return self.userName
    
    def getBalance(self):
        return self.balance
    
    def setPlan(self, plan):
        self.plan = plan

    def updateBalanceForVoiceCall(self, durationInSecs, destinationCircle):
        self.balance -= self.plan.calculateCallCost(durationInSecs, self.__getDistanceType(destinationCircle))
        return self.balance
    
    def updateBalanceForSMS(self, destinationCircle):
        self.balance -= self.plan.calculateSMSCost(self.__getDistanceType(destinationCircle))
        return self.balance;
    
    def updateBalance(self, rechargeAmount):
        self.balance += rechargeAmount

    def __getDistanceType(self, destinationCircle):
        return DistanceType().LOCAL if destinationCircle == self.circle else DistanceType().STD
