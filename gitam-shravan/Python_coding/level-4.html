Fundamentals - DRY: Level 4

With the report formatting out of the way, you can now start with actually building the report. The first kind of report you will build is a warehouse report. As shown in the diagram below, a warehouse simply stores one or more units of many different SKUs Once a month, the data team takes a snapshot of the warehouse contents. To access the snapshot data, you can utilize methods of the WarehouseSnapshotDao base class.

Sample code to fetch the WarehouseSnapshot for a given warehouse name, year and month

self.warehouseSnapshotDao.findOne((
        self.warehouseSnapshotDao.createEqualsQueryFilter(
                "name", warehouseName),
        self.warehouseSnapshotDao.createEqualsQueryFilter(
                "year", year),
        self.warehouseSnapshotDao.createEqualsQueryFilter(
                "month", month)))

To generate the report for a particular month, you need to populate an instance of WarehouseReport The four fields in the report have the following meaning:

  • units The total number of units (across all SKUs) in the warehouse
  • skus The number of SKUs in the warehouse
  • value The total value of units contained in the warehouse
  • changeInValue The change in total value for this warehouse when compared its value for the previous month

If you can't find all data necessary to generate the report, return None
